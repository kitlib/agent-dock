# AgentDock 原型设计文档

## 1. 目标
本原型文档用于描述 AgentDock 多 Agent 资源管理系统的页面结构、布局分区、核心交互和主要操作路径，帮助在正式开发前统一产品界面认知。

系统主目标：
- 以 Agent 为核心管理 Skills、MCP、Subagents
- 让用户快速查看、启停、分组、拖拽和批量管理资源
- Marketplace 作为辅助导入入口

---

## 2. 整体导航原型

建议一级导航：
- Agents
- Resources
- Marketplace
- Settings

说明：
- `Agents` 是主工作台
- `Resources` 是本地资源池视图
- `Marketplace` 是辅助导入页
- `Settings` 是系统配置页

---

## 3. Agents 页面原型

## 3.1 页面定位
Agents 页面是整个系统的核心页面，用户的大部分日常操作应在这里完成。

主要任务：
- 切换 Agent
- 查看当前 Agent 绑定的资源
- 启用 / 停用资源
- 分组管理
- 拖拽编排
- 批量操作

---

## 3.2 页面布局
建议采用三栏布局：

```txt
┌────────────────────────────────────────────────────────────────────┐
│ 顶部栏：页面标题 / 搜索 / 状态筛选 / 批量操作 / 添加资源          │
├───────────┬──────────────────────────────────────┬─────────────────┤
│ 左栏      │ 中间主区域                           │ 右栏            │
│ Agent图标 │ 顶部切换：Skills / MCP / Subagents   │ 详情面板        │
│ 或Agent表 │ 列表区：未分组列表 / 分组列表        │                 │
│           │ 支持多选 / 启停 / 拖拽               │                 │
└───────────┴──────────────────────────────────────┴─────────────────┘
```

补充规则：
- 左栏与中间区域之间的分割线支持左右拖拽调整宽度
- 中间区域与右栏之间的分割线支持左右拖拽调整宽度
- 调整后的宽度应在当前会话中保持
- 建议记住用户最近一次布局宽度

---

## 3.3 左栏：Agent 列表
左栏支持两种状态：

### 展开状态
每个 Agent 条目显示：
- 图标
- 名称
- 简要统计（如资源数 / 启用数）
- 当前选中状态
- 更多操作入口

### 收窄状态
只显示：
- Agent 图标
- 当前选中状态

补充规则：
- hover 显示 Agent 名称
- 点击图标切换 Agent
- 提供展开 / 收起按钮

左栏顶部或底部提供：
- 新建 Agent 按钮

每个 Agent 支持：
- 选中切换
- 重命名
- 删除

建议采用紧凑列表，不使用大卡片。

---

## 3.4 顶部工具栏
建议包含：
- 搜索框
- 状态筛选：All / Enabled / Disabled
- 分组筛选
- 批量操作按钮
- 添加资源按钮

目标：
- 让用户快速缩小当前视图范围
- 快速进入批量管理

---

## 3.5 中间主区域顶部切换
中间主区域顶部需要提供资源类型切换，用于切换显示：
- Skills
- MCP
- Subagents

交互规则：
- 一次只展示一种资源类型
- 切换后保留当前 Agent 不变
- 每种类型分别显示自己的未分组列表和分组列表
- 搜索、状态筛选、批量操作作用于当前选中的资源类型

这样可以减少单屏信息量，提高列表浏览效率。

---

## 3.6 中间主区域：资源编排区
建议使用紧凑列表，而不是卡片。

在当前资源类型下，分为两部分：

### 未分组区
用于放置尚未分组的资源。

### 分组区
每个分组显示：
- 分组名称
- 折叠/展开按钮
- 分组启用/停用入口
- 分组更多操作入口

每个资源列表项显示：
- 勾选框
- 名称
- 启用状态
- 简短描述
- 快捷操作

说明：
- 整个资源列表项都支持拖拽
- 不单独提供拖拽手柄

说明：
- 当前切到 Skills 时，只显示当前 Agent 的 Skills 列表
- 当前切到 MCP 时，只显示当前 Agent 的 MCP 列表
- 当前切到 Subagents 时，只显示当前 Agent 的 Subagents 列表

---

## 3.7 资源列表项原型
建议采用紧凑列表，两行显示：

```txt
[ ] Resource Name    Enabled    [More]
    Short description
```

如果需要补充类型信息，可在列表标题或顶部切换中体现，不必在每一行重复强调。

支持交互：
- 单击选中查看详情
- 勾选进入批量操作
- 点击开关启用/停用
- 直接拖拽整个列表项进行排序或跨组移动
- 更多菜单中执行移除、移动分组等操作

说明：
- 默认按紧凑列表行展示
- 第一行展示主信息与操作
- 第二行展示描述信息
- 保持高信息密度
- 适合快速浏览和批量处理

---

## 3.8 右栏：详情面板
当选中某个资源绑定时，右栏显示详情。

建议内容：
- 资源名称
- 当前资源类型
- 描述
- 当前 Agent 绑定状态
- 启用状态
- 所属分组
- 来源
- 本地路径（高级模式）
- 快捷操作按钮
- 内容预览区

快捷操作包括：
- 启用 / 停用
- 移动到分组
- 从 Agent 移除
- 打开更多信息

### 右栏按资源类型区分展示

#### Skills
右栏可浏览 Skill 的 Markdown 内容，适合展示：
- README
- 使用说明
- 参数说明
- 示例内容

建议界面：
- 上部显示基础信息与操作
- 下部显示 Markdown 预览区

#### MCP
右栏可浏览 MCP 的文档内容，适合展示：
- 接入说明
- 配置说明
- 参数说明
- 使用限制

建议界面：
- 上部显示基础信息与操作
- 下部显示文档预览区
- 可补充配置摘要区域

#### Subagents
右栏可浏览 Subagent 的内容，适合展示：
- 角色说明
- system prompt 摘要
- 能力边界
- 依赖资源说明

建议界面：
- 上部显示基础信息与操作
- 下部显示内容预览区
- 可补充依赖摘要区域

---

## 3.9 批量操作原型
当用户选中多个资源后，顶部进入批量模式。

建议显示：
- 已选数量
- 批量启用
- 批量停用
- 批量移动到分组
- 批量移出分组
- 批量移除绑定

如果支持批量删除本地资源，必须与“移除绑定”分开，并增加明显确认。

---

## 4. Resources 页面原型

## 4.1 页面定位
Resources 页面用于从全局视角查看本地资源池，而不是只看某个 Agent。

主要任务：
- 查看本地所有资源
- 搜索和筛选
- 查看资源详情
- 删除资源
- 导入资源
- 查看资源被哪些 Agent 使用

---

## 4.2 页面布局
建议采用两栏布局：

```txt
┌──────────────────────────────────────────────────────┐
│ 顶部栏：类型切换 / 搜索 / 导入按钮 / Marketplace 入口 │
├───────────────────────────────┬──────────────────────┤
│ 左侧资源列表                  │ 右侧详情面板         │
└───────────────────────────────┴──────────────────────┘
```

---

## 4.3 顶部栏
建议包含：
- 类型切换：Skills / MCP / Subagents
- 搜索框
- 导入按钮
- 前往 Marketplace 按钮

---

## 4.4 资源列表项
每项建议显示：
- 名称
- 来源
- 版本
- 是否启用
- 被多少个 Agent 使用

列表支持：
- 点击查看详情
- 删除资源
- 快速跳转到使用该资源的 Agent

建议使用紧凑列表行，不使用卡片。

示例：

```txt
Resource Name    Local    v1.0.0    Enabled    Used by 3 agents
```

---

## 4.5 详情面板
建议显示：
- 基础信息
- 当前资源类型
- 资源来源
- 本地路径
- 状态信息
- 被哪些 Agent 使用
- 删除按钮

如果资源未被任何 Agent 使用，可强调“可安全清理”。

---

## 5. Marketplace 页面原型

## 5.1 页面定位
Marketplace 页面是辅助导入入口，用于发现和安装资源。

主要任务：
- 浏览资源
- 搜索资源
- 查看资源详情
- 安装到本地资源池
- 安装后可立即绑定到 Agent

---

## 5.2 页面布局
建议采用两栏布局：

```txt
┌─────────────────────────────────────────────────────────────┐
│ 顶部栏：搜索 / source 选择 / 状态 / source 设置入口         │
├────────────────────────────────┬────────────────────────────┤
│ 左侧市场资源列表               │ 右侧详情与安装面板         │
└────────────────────────────────┴────────────────────────────┘
```

---

## 5.3 顶部栏
建议包含：
- 搜索框
- 资源类型切换：Skills / MCP / Subagents
- source 选择器
- provider 状态提示
- 打开 source 设置按钮

首期重点支持 Skills，MCP 和 Subagents 可先作为预留入口。

---

## 5.4 列表项
建议显示：
- 资源名称
- 描述
- 来源
- 版本
- 安装状态
- 安装按钮

状态示例：
- 未安装
- 已安装
- 可更新

建议采用紧凑列表形式，例如：

```txt
Resource Name    Skillhub    v1.2.0    Not installed    [Install]
```

---

## 5.5 详情与安装面板
建议显示：
- 名称
- 当前资源类型
- 描述
- 来源
- 版本
- 说明信息
- 安装按钮

安装完成后提供：
- 绑定到 Agent 选择器
- 或“稍后绑定”选项

---

## 5.6 Source 设置原型
可采用 Dialog 或侧边面板。

建议显示：
- source 列表
- source 名称
- provider
- endpoint
- enabled 状态
- health 状态

支持操作：
- 新增
- 编辑
- 删除
- 启用 / 禁用
- 健康检查

建议使用紧凑表格或列表。

## 6. 关键交互原型

## 6.1 新建 Agent
路径：
1. 在 Agents 页点击“新建 Agent”
2. 填写名称
3. 创建完成后自动进入该 Agent
4. 中间区域为空，提示添加资源

---

## 6.2 为 Agent 添加资源
路径：
1. 在 Agents 页点击“添加资源”
2. 打开资源选择面板
3. 从当前资源类型的本地资源池中选择一个或多个资源
4. 添加到当前 Agent
5. 新资源进入当前类型下的未分组区

---

## 6.3 切换资源类型
路径：
1. 在中间主区域顶部点击 Skills / MCP / Subagents
2. 页面只展示当前 Agent 下该类型资源
3. 保留当前 Agent 不变
4. 列表、分组、批量操作同步切换到当前类型

---

## 6.4 拖拽到分组
路径：
1. 用户直接拖拽整个资源列表项
2. 拖到目标分组区域
3. 放开后更新顺序与分组
4. UI 立即反馈最新位置

---

## 6.5 批量启停
路径：
1. 用户勾选多个资源
2. 顶部出现批量操作栏
3. 点击“批量启用”或“批量停用”
4. 统一更新状态

---

## 6.6 从 Marketplace 安装并绑定
路径：
1. 打开 Marketplace
2. 搜索并选择资源
3. 点击安装
4. 安装完成后弹出“绑定到 Agent”选择
5. 选择某个 Agent 或跳过

---

## 7. 状态与反馈

### 7.1 空状态
需要覆盖：
- 没有 Agent
- 当前 Agent 在当前类型下没有资源
- 当前没有分组
- 本地资源池为空
- Marketplace 没有搜索结果

### 7.2 操作反馈
建议统一使用：
- toast 提示
- 列表内状态更新
- 详情面板状态更新

### 7.4 交互约束
为避免实际使用中的歧义，补充以下交互约束：

#### 分组作用域
- 分组仅作用于“当前 Agent + 当前资源类型”
- Skills、MCP、Subagents 各自维护独立分组
- 切换资源类型后，不共享分组结构

#### 启停作用域
- 主工作流中的启停指“当前 Agent 下的资源绑定启停”
- 不将 Resources 页中的资源状态与 Agent 绑定启停混为一体
- 删除绑定与删除资源本体必须明确区分

#### 拖拽规则
- 整个资源列表项支持拖拽
- 勾选框、开关、更多菜单等控件区域不触发拖拽
- 拖拽只作用于当前 Agent、当前资源类型下的列表

#### 多选规则
- 多选后进入批量操作模式
- 首期多选后不直接做整体拖拽
- 多选移动分组通过批量操作完成

#### 右栏展示规则
- 右栏按资源类型展示不同内容
- Skills 展示 Markdown 内容
- MCP 展示文档与配置摘要
- Subagents 展示角色内容与依赖摘要
- 右栏建议分为概览区与内容区，避免信息过重

#### 左栏收窄规则
- 左栏收窄后仅显示 Agent 图标
- hover 显示 Agent 名称
- 当前选中 Agent 需要明显高亮

---

## 8. 原型原则

- Agents 页面使用紧凑列表作为主展示形式
- Resources 页面使用紧凑列表作为主展示形式
- Marketplace 页面使用紧凑列表作为主展示形式
- 不使用大卡片布局
- 左栏支持收窄为仅显示 Agent 图标
- 左栏与中间区域之间的分割线支持左右拖拽调整宽度
- 中间区域与右栏之间的分割线支持左右拖拽调整宽度
- 中间主区域顶部支持切换 Skills / MCP / Subagents
- 主工作流集中在 Agents 页面
- Marketplace 只负责导入，不承担主编排
- 绑定删除与资源删除必须明确区分
- 拖拽和批量操作应尽量直观
- 高风险操作要有确认
- MCP 敏感配置默认不明文展示

---

## 9. 原型结论
本系统的原型应以“多 Agent 本地资源管理”为核心，以 Agents 页面作为主工作台，通过 Resources 页面提供全局资源视角，通过 Marketplace 页面补充资源导入能力。整体界面应优先服务高频操作：查看、启停、分组、拖拽和批量管理，默认采用紧凑列表形式而不是卡片形式；同时左栏支持收窄为仅显示 Agent 图标，中间主区域顶部支持切换 Skills、MCP 和 Subagents，以便用户更高效地管理不同类型的资源。
