# AgentDock 多 Agent 资源管理功能设计

## 功能定位

AgentDock 的核心目标是成为一个面向多 Agent 的本地资源管理中心。

当前产品重点：
- 管理本地 Agent
- 浏览和维护本地 Skills
- 在统一工作区内承载 Skills、MCP、Subagents 三类资源
- 通过 Marketplace 作为辅助导入入口

说明：
- `Home / Agents` 仍然是当前主工作台
- `Resources` 页面仍是规划项，尚未正式落地
- `Marketplace` 目前仍以占位能力为主，真实接入尚未完成

## 当前实现现状（2026-04-16）

### 1. Agents 工作台

当前已实现：
- 左侧 Agent 列表支持折叠
- 左侧顶部支持“全部”入口
- 选中“全部”时，可在 Home 内查看聚合后的本地 Skills
- 选中单个 Agent 时，继续查看该 Agent 的本地资源

说明：
- “全部”当前仅覆盖 Skills
- MCP / Subagents 暂未并入同一套全局视图

### 2. Skill 管理

当前已实现：
- 本地 Skill 扫描、列表、详情
- 启用 / 停用
- 打开 Skill 目录
- 打开 Skill 入口文件
- 删除本地 Skill
- 批量复制 Skill 到其它 Agent
- 单个 Skill 从“更多”菜单复制到其它 Agent

### 3. Skill 复制交互

当前已实现：
- 复制目标使用平铺 Agent 卡片，而不是下拉框
- 支持多选目标 Agent
- 支持冲突预览
- 支持 overwrite / skip 决议
- 关闭弹窗后会清理预览状态，避免残留

### 4. 列表与详情操作顺序

当前规则：
- 列表“更多”菜单：打开、编辑、复制、停用/启用、删除
- 右栏详情操作：打开、编辑、停用/启用、删除

设计原则：
- 高频非破坏操作优先
- 危险操作后置

## 核心功能结构

### 1. 多 Agent 管理

支持：
- Agent 列表展示
- 新增 Agent
- 删除 Agent
- 切换当前 Agent
- 查看每个 Agent 的资源概览

### 2. 本地资源池管理

统一管理本地资源：
- Skills
- MCP
- Subagents

目标能力：
- 查看资源列表
- 查看资源详情
- 查看启用状态
- 删除资源
- 从本地导入资源
- 查看资源与 Agent 的关系

### 3. Agent 资源绑定管理

支持：
- 向 Agent 添加资源
- 从 Agent 移除资源
- 启用 / 禁用绑定资源
- 调整顺序
- 查看绑定状态

### 4. 分组管理

规划支持：
- 创建分组
- 重命名分组
- 删除分组
- 折叠 / 展开分组
- 分组排序

### 5. 拖拽编排

规划支持：
- 未分组区内排序
- 分组内排序
- 从未分组拖入分组
- 跨组拖拽移动

### 6. 批量操作

规划支持：
- 批量启用
- 批量停用
- 批量移动到分组
- 批量移出分组
- 批量解除绑定
- 可选的批量删除本地资源

### 7. Marketplace 辅助导入

规划支持：
- 浏览市场资源
- 搜索资源
- 查看资源详情
- 从 Marketplace 安装资源到本地资源池
- 安装后绑定到某个 Agent，或稍后再绑定

## 页面结构

### Agents / Home

当前主工作台，用于：
- 切换 Agent
- 查看当前 Agent 的资源
- 查看“全部”聚合 Skills
- 执行启停、复制、删除等维护操作

### Resources

规划中的独立资源页，用于：
- 查看所有本地 Skills、MCP、Subagents
- 查看资源详情
- 查看资源关系
- 执行全局资源管理

### Marketplace

辅助导入页，用于：
- 浏览市场资源
- 搜索资源
- 查看详情
- 安装到本地资源池

## 首期边界

已落地：
- 多 Agent 基础管理
- 本地 Skill 扫描与展示
- Skill 启停、打开、删除、复制
- Home 内“全部 Skills”聚合视图

未完成：
- 独立 Resources 页面
- Marketplace 真接入
- Skill 使用关系视图
- 分组管理
- 拖拽编排
- MCP / Subagents 的真实全局管理链路

## 使用原则

- 日常主操作优先在 `Home / Agents` 中完成
- “全部”视图当前只用于聚合本地 Skills
- Marketplace 只负责资源发现和导入，不承担主编排流程
- 删除绑定与删除资源本体必须明确区分
- 高风险删除操作必须确认
- 危险操作在视觉顺序上后置
