# Commit Convention

## Scope

This repository uses a lightweight Conventional Commits style for normal development commits.

## Format

Use:

```text
type(scope): summary
```

Guidelines:

- Prefer `feat`, `fix`, `refactor`, `docs`, `chore`, `test`
- Keep `summary` imperative and concise
- Add `scope` when it helps explain the affected area

Common scopes in this repository:

- `mcp`
- `skill`
- `marketplace`
- `agent`
- `ui`

## Examples

```text
feat(mcp): implement local MCP management and JSON import
fix(skill): refresh local skill list after delete
docs(mcp): update local management status
```

## Release Commit

Release commits are generated by `pnpm release:version` and should use:

```text
chore: release x.y.z
```
